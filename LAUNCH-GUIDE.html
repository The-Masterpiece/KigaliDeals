1
Set up Supabase — your backend database
PostgreSQL database, authentication, and real-time API in one
Free tier available ▾
1
Create your Supabase project
Go to supabase.com → New Project → name it "kigalideals" → select region "Europe West" (closest to Rwanda) → choose a strong database password.

→ Create Project
2
Run the database schema
In your Supabase dashboard → SQL Editor → New Query. Paste the entire contents of schema.sql and click Run. This creates all tables, security policies, and functions in one shot.
What gets created: profiles, businesses, deals, vouchers, payments, payouts, reviews tables — all with row-level security so users only see their own data.
3
Copy your API keys
Settings → API. Copy these into your .env.local file: 
NEXT_PUBLIC_SUPABASE_URL=https://xxxx.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbGciO...
SUPABASE_SERVICE_ROLE_KEY=eyJhbGciO... # Keep secret!
⚠️ Never expose SUPABASE_SERVICE_ROLE_KEY to the browser. Only use it in server-side API routes.
4
Set up authentication
Authentication → Providers → Email: enable. Under URL Configuration, set:
• Site URL: https://kigalideals.rw
• Redirect URLs: https://kigalideals.rw/auth/callback

Optionally enable Google OAuth for easier login — Auth → Providers → Google.
5
Enable Supabase Storage for images
Storage → New bucket → name: deal-images → Public: ON. Create a second bucket: business-logos → Public: ON.

This is where business logos and deal photos will be stored.
  • Supabase project created
  • schema.sql executed successfully (no errors)
  • API keys copied to .env.local
  • Auth redirect URLs configured
  • Storage buckets created
2
Integrate MTN Mobile Money payments
Accept payments from every MTN and Airtel subscriber in Rwanda
Revenue critical ▾
1
Register at MTN MoMo Developer Portal
Go to momodeveloper.mtn.com → Sign Up → Create an account.
Subscribe to the Collection product (customer pays you) and the Disbursements product (you pay businesses).

→ Developer Portal
2
Provision sandbox API credentials
In the developer portal → My Apps → your app → Generate API User and API Key. These go in your .env.local: 
MOMO_SUBSCRIPTION_KEY=abc123...
MOMO_API_USER=uuid-from-portal
MOMO_API_KEY=your-api-key
MOMO_ENVIRONMENT=sandbox
Sandbox mode: You can test with any fake phone number. No real money moves. When ready for production, change MOMO_ENVIRONMENT to "production" and use real credentials.
3
Create the API routes in Next.js
Create these two files in your Next.js app: 
# pages/api/payments/initiate.js
# → Customer clicks "Pay" → this triggers MoMo prompt on their phone

# pages/api/momo/callback.js
# → MoMo calls this URL when payment succeeds/fails
# → Creates the voucher and sends WhatsApp confirmation
The complete code for both is inside lib/momo.js — look for the comment blocks at the bottom.
4
Test a sandbox payment end-to-end
# Install dependencies
npm install

# Start dev server
npm run dev

# Test payment initiation via curl
curl -X POST http://localhost:3000/api/payments/initiate \
  -H "Content-Type: application/json" \
  -d '{"dealId":"...","quantity":1,"phoneNumber":"0780000000","totalAmount":15000}'
In sandbox, any 10-digit number works. Expected response: {"success":true,"referenceId":"uuid"}
5
Apply for production access (takes ~2 weeks)
MTN requires your business to be registered in Rwanda (RDB) before they grant production API access. Start this process early:
• Submit your RDB certificate of incorporation
• Your company bank account details
• A demo of your platform working in sandbox

Contact MTN Rwanda's fintech team: fintech@mtn.co.rw
  • Registered at momodeveloper.mtn.com
  • Subscribed to Collection and Disbursements
  • Sandbox API keys in .env.local
  • Sandbox payment test successful
  • Production application submitted to MTN
3
Launch your WhatsApp bot + daily deal alerts
Automated voucher delivery, daily blasts, and expiry reminders
Growth engine ▾
1
Set up Twilio WhatsApp Sandbox (live in 5 minutes)
twilio.com → Sign Up (free) → Console → Messaging → Try it Out → WhatsApp.
You'll get a sandbox number. To test, save +1 415 523 8886 on your phone and send the join code they give you. 
TWILIO_ACCOUNT_SID=ACxxxxxxxx
TWILIO_AUTH_TOKEN=your_token
TWILIO_WHATSAPP_FROM=whatsapp:+14155238886
2
Set up the inbound webhook
In Twilio Console → WhatsApp Sandbox → Webhook URL, set:
https://kigalideals.rw/api/whatsapp/webhook

Create pages/api/whatsapp/webhook.js and use the handleInboundWhatsApp function from lib/whatsapp.js.
What the bot handles: DEALS, FOOD, SPA, GYM, HOTEL commands + VOUCHER [code] lookup + STOP to unsubscribe. You can add more commands anytime.
3
Set up the daily deals cron job
On Vercel, use Vercel Cron (free on Pro plan) or use cron-job.org (free) to call your API every day at 8am Kigali time: 
# vercel.json — add cron configuration
{
  "crons": [
    {
      "path": "/api/cron/daily-deals",
      "schedule": "0 6 * * *"  // 6am UTC = 8am Kigali
    },
    {
      "path": "/api/cron/expiry-reminders",
      "schedule": "0 7 * * *"
    },
    {
      "path": "/api/cron/weekly-payouts",
      "schedule": "0 6 * * 1"  // Every Monday
    }
  ]
}
4
Apply for a real WhatsApp Business number
For production (sending to any number, not just sandbox):
• Apply via business.whatsapp.com
• Or apply through Twilio directly (faster, ~1-2 weeks)
• You need a Meta Business account and a Facebook page

Once approved, update TWILIO_WHATSAPP_FROM to your real number.
  • Twilio account created and sandbox working
  • Webhook URL configured in Twilio
  • Bot responds correctly to DEALS, FOOD, STOP commands
  • Voucher confirmation sends after successful payment
  • Daily cron job scheduled and tested
4
Deploy to Vercel + connect kigalideals.rw
From your computer to live on the internet in under 10 minutes
Free hosting ▾
1
Push your code to GitHub
# One-time setup
git init
git add .
git commit -m "Initial commit — KigaliDeals"

# Create a new repo on github.com, then:
git remote add origin https://github.com/yourname/kigalideals.git
git push -u origin main
Make sure .env.local is in your .gitignore — never push secret keys!
2
Deploy to Vercel in one click
vercel.com → Sign up with GitHub → New Project → Import your kigalideals repo → Deploy.

Vercel automatically detects Next.js and builds correctly. Your site will be live at yourname.vercel.app within 2 minutes.

→ Deploy to Vercel
3
Add all environment variables to Vercel
Vercel Dashboard → your project → Settings → Environment Variables. Add every variable from your .env.example file. Make sure to mark SUPABASE_SERVICE_ROLE_KEY and all secret keys as Production only.
Critical: After adding env vars, you must redeploy. Click Deployments → Redeploy on your latest deployment.
4
Connect your kigalideals.rw domain
Buy domain at RICTA (Rwanda) or Namecheap.

In Vercel → Settings → Domains → Add Domain → type kigalideals.rw.
Vercel gives you two DNS records. Add them to your domain registrar: 
# Add these DNS records at your registrar:
A Record    @    76.76.19.19
CNAME      www  cname.vercel-dns.com
DNS propagates in 5–60 minutes. SSL certificate is automatic (free).
5
Final production checklist before launch
Run through this before announcing to anyone:
  • Code pushed to GitHub (no secrets in repo)
  • Deployed successfully on Vercel
  • All env vars added to Vercel and redeployed
  • kigalideals.rw domain connected with SSL
  • Supabase auth redirect URL updated to production domain
  • MoMo callback URL updated to production domain
  • WhatsApp webhook URL updated to production domain
  • End-to-end test: register → browse → buy → receive WhatsApp
  • First 10 real deals listed from actual Kigali businesses
  • Admin has been tested and can approve/reject deals
